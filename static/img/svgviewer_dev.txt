Docusaurus မှာ **`shop`** ဆိုတဲ့ folder အသစ်ကို `docs` နဲ့ `blog` လိုမျိုး အလုပ်လုပ်အောင် လုပ်ဖို့အတွက် **Multi-instance plugin** ကို သုံးရပါမယ်။ သင်က config ထဲမှာ link တွေ ထည့်ထားပြီးသားဖြစ်ပေမဲ့၊ Docusaurus က `shop` folder ကို ဘယ်လို process လုပ်ရမလဲဆိုတာ မသိသေးလို့ Error တက်နေတာပါ။

အောက်ပါအဆင့်တွေအတိုင်း ပြင်လိုက်ပါ-

### အဆင့် (၁) `docusaurus.config.js` ကို ပြင်ဆင်ခြင်း

သင့်ရဲ့ `presets` အောက်က `docs` setting အပြီးမှာ `plugins` ဆိုတဲ့ section တစ်ခုထပ်တိုးပြီး `shop` folder ကို document instance တစ်ခုအနေနဲ့ သတ်မှတ်ပေးရပါမယ်။

```javascript
// ... existing code ...
  presets: [
    [
      'classic',
      /** @type {import('@docusaurus/preset-classic').Options} */
      ({
        docs: {
          sidebarPath: './sidebars.js',
          // ...
        },
        blog: {
          // ...
        },
        theme: {
          customCss: './src/css/custom.css',
        },
      }),
    ],
  ],

  // ဒီအပိုင်းကို ထပ်ထည့်ပေးပါ
  plugins: [
    [
      '@docusaurus/plugin-content-docs',
      {
        id: 'shop', // instance id
        path: 'shop', // folder name
        routeBasePath: 'shop', // URL path
        sidebarPath: './sidebars.js', // သို့မဟုတ် sidebar အသစ်သုံးနိုင်သည်
      },
    ],
  ],

  themeConfig:
// ... rest of your config ...

```

---

### အဆင့် (၂) `sidebars.js` ကို ပြင်ဆင်ခြင်း

`shop` folder အတွက် sidebar content တွေကို auto-generate လုပ်ပေးဖို့ sidebar config ထဲမှာ id တစ်ခု ထပ်တိုးပေးရပါမယ်။

```javascript
const sidebars = {
  // နဂိုရှိပြီးသား docs sidebar
  tutorialSidebar: [{type: 'autogenerated', dirName: '.'}],

  // Shop အတွက် အသစ်ထည့်ရန်
  shopSidebar: [{type: 'autogenerated', dirName: '.'}], 
};

export default sidebars;

```

---

### အဆင့် (၃) Navbar Item ကို ပြင်ဆင်ခြင်း

သင် config ထဲမှာရေးထားတဲ့ `{to: '/shop', label: 'Shop', ...}` က ရိုးရိုး link ဖြစ်နေပါတယ်။ အကယ်၍ `shop` ထဲက sidebar ကိုပါ မြင်ချင်တယ်ဆိုရင် navbar item ကို အခုလို ပြင်ရေးပေးပါ။

```javascript
// themeConfig.navbar.items ထဲမှာ ပြင်ရန်
{
  type: 'docSidebar',
  sidebarId: 'shopSidebar', // sidebars.js ထဲက id နဲ့ တူရမယ်
  docsPluginId: 'shop',     // plugins ထဲက id နဲ့ တူရမယ်
  position: 'left',
  label: 'Shop',
},

```

---

### မြန်မာလို ရှင်းပြချက် (Explanation)

1. **Plugins Section**: Docusaurus မှာ folder အသစ်တစ်ခုကို docs ပုံစံမျိုး သုံးချင်ရင် `plugin-content-docs` ကို instance တစ်ခုအနေနဲ့ ထပ်တိုးပေးရပါတယ်။ ဒါမှ Docusaurus က `/shop` ဆိုတဲ့ route ကို သိမှာပါ။
2. **id**: `id: 'shop'` လို့ ပေးလိုက်တာက ဒီ document instance ကို တခြားဟာတွေနဲ့ မမှားအောင် နာမည်ပေးလိုက်တာပါ။
3. **Navbar**: `type: 'docSidebar'` ကို သုံးလိုက်ခြင်းအားဖြင့် User က Shop menu ကို နှိပ်လိုက်ရင် `shop` folder ထဲက content တွေနဲ့ သက်ဆိုင်တဲ့ sidebar ကို ဘေးမှာ မြင်ရမှာ ဖြစ်ပါတယ်။

**နောက်ထပ် ဘာလုပ်ပေးရမလဲ?**
အခုပြင်လိုက်တဲ့ setting တွေ အလုပ်လုပ်ဖို့အတွက် Terminal မှာ `npm start` ကို ပိတ်ပြီး ပြန်ဖွင့်ပေးဖို့ မမေ့ပါနဲ့။ `shop/intro.md` ထဲမှာ content အနည်းငယ် ရှိနေဖို့လည်း လိုအပ်ပါတယ်။

Sidebars တွေကို shop အတွက် သီးသန့် file တစ်ခု ခွဲထုတ်ချင်ပါသလား? သို့မဟုတ် ဒီအတိုင်းပဲ အဆင်ပြေပါသလား?
